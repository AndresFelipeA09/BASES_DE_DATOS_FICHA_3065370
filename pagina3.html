3.4. Lenguajes de los sistemas administradores de bases de datos.

Definicion:

Los lenguajes de bases de datos son componentes de los sistemas gestores de bases de datos (SGBD). los cuales permiten crear, actualizar, eliminar, consultar y analizar datos de una base de datos

![image alt](https://github.com/Dalton-Mar/BASES_DE_DATOS_FICHA_3065370/blob/main/manejadores%20de%20base%20de%20datos.jpg?raw=true)

Los lenguajes de bases de datos se clasifican en:

Lenguaje de Definición de datos (DDL): es un subconjunto del lenguaje de consulta estructurado (SQL) que se utiliza para definir y modificar la estructura de una base de datos

¿Que se puede hacer con el DDL?

  • Crear, modificar o eliminar objetos de base de datos
  • Definir relaciones de integridad referencial
  • Configurar un sistema de prueba
  • Generar plantillas para nuevos objetos


Lenguaje de Manipulación de Datos (DML): es un conjunto de comandos que se usan para modificar, agregar, eliminar y consultar datos en bases de datos. Es un sublenguaje de un lenguaje de base de datos más amplio, como SQL.

Los principales comandos DML son:

  • SELECT: Se utiliza para consultar datos
  • INSERT: Se utiliza para insertar valores en una base de datos
  • UPDATE: Se utiliza para modificar los valores de uno o varios registros
  • DELETE: Se utiliza para eliminar las filas de una tabla


Lenguaje de control de datos (DCL): es un conjunto de comandos SQL que permite controlar el acceso a los datos de una base de datos. Es un componente del Lenguaje de Consulta Estructurado

es util para:

  • Otorgar o revocar privilegios a los usuarios de una base de datos
  • Gestionar el acceso, los privilegios y la seguridad de los usuarios
  • Crear aplicaciones con mecanismos sólidos de protección de datos
  • Mantener el cumplimiento normativo


Lenguaje de Control de Transacciones (TCL):es un conjunto de comandos que se usan para gestionar las transacciones en una base de datos. Es un subconjunto del lenguaje SQL.

funciones del TCL:

  • Supervisar, controlar y gestionar las transacciones en la base de datos
  • Mantener la consistencia de la base de datos
  • Realizar un seguimiento de las modificaciones que realizan las sentencias DML
  • Organizar las sentencias en transacciones lógicas
