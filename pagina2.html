Modelo Entidad-Relación

¿Qué es el Modelo Entidad-Relación?

El Modelo Entidad-Relación (E-R) es una herramienta para representar cómo personas, objetos o conceptos se relacionan dentro de una base de datos. Fue propuesto por Peter Chen en 1976.

Ejemplo de modelo E-R con entidades y relaciones.

Elementos del Modelo

  • Entidades: Representan objetos del mundo real (Ej: Cliente, Producto, Aprendiz).
  • Atributos: Características de las entidades (Ej: Nombre, Dirección).
  • Clave primaria: Identificador único de cada entidad.
  • Relaciones: Conexiones entre entidades (Ej: Cliente compra Producto).

Ejemplo de relaciones entre entidades.

Tipos de Relaciones

  • Uno a Uno: Cada entidad solo se conecta con otra (Ej: Persona - Pasaporte).
  • Uno a Muchos: Una entidad se relaciona con varias (Ej: Cliente - Pedidos).
  • Muchos a Muchos: Varias entidades pueden vincularse entre sí (Ej: Estudiantes - Cursos).

Ejemplo de relación Uno a Muchos.

Usos del Modelo

  • Diseño de bases de datos.
  • Optimización de procesos empresariales.
  • Análisis de datos en investigaciones.
  • Educación y planificación de sistemas de información.
