1. Conceptos Generales de Bases de Datos

Una base de datos es una herramienta para recopilar y organizar información. Las bases de datos pueden almacenar información sobre personas, productos, pedidos u otras cosas.
Muchas bases de datos comienzan como una lista en una hoja de cálculo o en un programa de procesamiento de texto.
A medida que la lista aumenta su tamaño, empiezan a aparecer redundancias e inconsistencias en los datos. Cada vez es más difícil comprender los datos en forma de lista
y los métodos de búsqueda o extracción de subconjuntos de datos para revisión son limitados.
Una vez que estos problemas empiezan a aparecer, es una buena idea transferir los datos a una base de datos creada por un sistema de administración de bases de datos (DBMS), como Access.
Una base de datos computarizada es un contenedor de objetos. Una base de datos puede contener más de una tabla. Por ejemplo, un sistema de seguimiento de inventario
que usa tres tablas no son tres bases de datos, sino una base de datos que contiene tres tablas.
A menos que se haya diseñado específicamente para usar datos o código de otro origen, una base de datos de Access almacena sus tablas en un único archivo,
junto con otros objetos, como formularios, informes, macros y módulos. Las bases de datos creadas en el formato Access 2007-2016
(que también usa Access para Microsoft 365, Access 2024, Access 2021 y Access 2019 ) tienen la extensión de archivo .accdb, y las bases de datos creadas
en formatos de Access anteriores tienen la extensión de archivo .mdb.

1.2. Tipos de bases de datos.

Los tipos de Base de Datos se clasifican segun su estrucutura y forma de almacenamiento: 1) Jerarquias: Organiza los datos en estrcutura de árbol con relaciones padre-hijo. 2) En Red: Permiten relaciones complejas entre datos mediante grafos. 3) Relaciones: Usan tablas con filas y columnas para estrcuturar datos y establecer relaciones (SQL). 4) Orientadas a Objetos: Almacenan datos como objetos con atributos y metodos, usados en programación OOP. 5) NoSQL: Bases de datos no relacionales que almacenan datos en documentos, clave-valor, columnas o grafos, ideales para grandes volumenes de información. 6) Distribuidas: Los Datos estan repartidos en multiples servidores para mejorar redundancia y rendimiento.

1.3. Clasificación de bases de datos. (2)

Las bases de datos se clasifican por modelos de datos (relacionales y NoSQL) y por distribucion (centralizadas, distribuidas y replicadas), segun como organizan, almacenan y gestionan la informacion.

Existen varios tipos de bases de datos según su estructura y uso. Las **bases de datos relacionales** (como MySQL o PostgreSQL) organizan la información en tablas y usan SQL para consultarla. Las **bases de datos no relacionales** o **NoSQL** (como MongoDB o Redis) permiten almacenar datos no estructurados en formatos como documentos, pares clave-valor, grafos o columnas. Las **bases de datos en memoria** (como Redis) ofrecen acceso muy rápido al guardar los datos directamente en la RAM. También están las **bases de datos orientadas a objetos**, que almacenan datos como objetos de programación, y las **bases de datos distribuidas**, que reparten los datos en múltiples servidores para mayor disponibilidad. Finalmente, las **jerárquicas** y **de red** son modelos más antiguos que organizan los datos como árboles o redes con relaciones complejas.

1.3. Clasificación de bases de datos. (2)

Las bases de datos se pueden clasificar según su modelo de datos, como relacionales (tablas y SQL), NoSQL (documentos, grafos, clave-valor o columnas), orientadas a objetos, jerárquicas y de red. También se clasifican por su ubicación, siendo centralizadas (todo en un solo lugar) o distribuidas (datos repartidos en varios nodos). Según su contenido, pueden ser bibliográficas, de texto completo, numéricas o multimedia. Otra clasificación es por su uso o acceso, como locales, en la nube, personales, corporativas, o de tiempo real, y también pueden diferenciarse por su persistencia, siendo temporales (en memoria) o persistentes (almacenadas a largo plazo).

1.4. Sistema de Gestión de bases de datos.

Un Sistema de Gestión de Bases de Datos (SGBD) es un conjunto de herramientas y software diseñados para facilitar la creación, manipulación, consulta y gestión de bases de datos. Su propósito principal es permitir que los usuarios y aplicaciones interactúen con los datos de manera eficiente, organizada y segura. Los SGBD se encargan de manejar la estructura de los datos, administrar los accesos, y garantizar su integridad y consistencia.
