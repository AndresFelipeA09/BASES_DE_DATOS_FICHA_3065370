Modelo Entidad Relación

Es una metodología para diseñar y representar bases de datos de una manera conceptual, se usa
para estructurar la información de un sistema de entidades, atributos y relaciones
Componentes principales del MER

Entidades: estas representas objetos u conceptos que tienen existencia en el mundo real que tienen una existencia
independiente a la base de datos, se clasifica en:

-Entidades fuertes: Son una Fuente totalmente independiente como Cliente y Proveedor -Entidades débiles: Depende de otra entidad para existir como “Detalles de factura” depende de “factura”

2.3. Relaciones de muchos a muchos.

En bases de datos relacionales, una relación de muchos a muchos (M:N) se da cuando múltiples registros de una tabla pueden estar asociados con múltiples registros de otra tabla. Este tipo de relación no puede ser representada directamente entre dos tablas, por lo que se necesita una tabla intermedia (también conocida como tabla puente o de relación) que contenga claves foráneas de ambas tablas relacionadas. Ejemplo:Imaginemos dos entidades: Estudiantes y Cursos. Un estudiante puede estar inscrito en varios cursos. Un curso puede tener varios estudiantes inscritos. https://sl.bing.net/dNJvaKmDWAC
